### Описание

__app__ - базовое приложение предназначено для загрузки __функциональных модулей__ из
динамических библиотек .so и выполнения их функционала.

Сам по себе __app__ умеет только:
- загружать динамические библиотеки .so
- выполнять команды из загруженных библиотек *.so
- запускать асинхронную среду __tokio__
- создавать соединение с бд __PostgreSQL__
- отправлять события в модули
- загружать настройки из конфигурационного файла, командной строки и переменных
  среды
- отображать загруженные в приложение настройки
- выводить отладочную информацию по работе модулей

*app* содержит модули:
1. __MOD_ASYNC__ - запуск асинхронной среды tokio
2. __MOD_DB__ - соединение с бд PostgreSQL
3. __MOD_LD__ - загрузка динамических библиотек *.so
4. __MOD_CMD__ - парсинг и выполнение команд из командной строки
5. __MOD_CONFIG__ - вывод загруженных настроек приложения
6. __MOD_ENV__ - загрузка переменных среды из .env
7. __MOD_LOG__ - вывод отладочной информации в консоль или файл лога
8. __MOD_HELP__ - вывод справки приложения

### Сборка

`make build-bin-app` - создание приложения app  
`make build-lib-app-example-b no_std=1` - (создает libapp_b.so) пример модуля *MOD_B* с *no_std*  
`make build-lib-app-example-a` - (создает libapp_a.so) пример асинхронного модуля *MOD_A* с работой бд
__(зависит от libapp_b.so)__  

### Команды

`target/release/app` - вывод справки программы *app*

`target/release/app -L app_a a` - загрузка библиотеки libapp_a.so и выполнение
команды *a* из модуля *MOD_A* (модуль *MOD_A* подзагрузит модуль *MOD_B* из
*libapp_b.so* автоматически так как прописана зависимость модуля *MOD_A* от *MOD_B*)

`target/release/app -L app_b b` - загрузка библиотеки libapp_b.so и выполнение
команды *b* из модуля *MOD_B*
