app-kernel/App
============

Название
--------

    App - базовый компонент для запуска модульного приложения.

Описание
--------

App предназначен для запуска модульного приложения.  
Основная работа App заключается в вызове событий у модулей.  

***События*** вызываемые App для модулей:

1. **LOAD**   - загрузка модуля в память
2. **UNLOAD** - выгрузка модуля из памяти
3. **NOTIFY** - отправка уведомления *(модули могут отправлять уведомления друг другу для выполнения полезной работы)*
4. **HOOK**   - вызов перехватчиков, которые вызывается до и после событий *LOAD,UNLOAD,NOTIFY*
5. **META**   - получение мета данных модуля

Алгоритм
--------

|           Стадия        | Описание |
| :---:                   | :---     |
| `App::new(&[MODULES..])`| Создание `App` со списком модулей и вызов события
|                         | `LOAD` у каждого модуля в переданном списке. 
|                         ||
| `App::run(<CONTEXT>)`   | Вызывает событие `NOTIFY` с уведомлением 
|                         | `NT_APP_RUN` у каждого подписавшегося модуля 
|                         | на это уведомление. |
|                         ||
| `App::drop()`           | Вызывает событие `UNLOAD` у каждого загруженного модуля. |


Обозначения
-----------

| Название          | Описание |
| :---:             | :---     |
| `Модуль`          | программный компонент предназначенный для инициализации и обработки событий.
|                   | Представляет из себя указатель на функцию принимающая события от компонента App.|
|                   ||
| `Мета данные`     | информация о модуле: адрес функции обработчика модуля,
|                   | название модуля, список зависимостей от других
|                   | модулей, список обрабатываемых событий и хуков, список
|                   | выполняемых команд |
|                   ||
| `Событие`         | числовое значение из *enum* перечисления
|                   ||
| `Уведомление`     | уникальный строковый идентификатор для события `NOTIFY`
|                   ||
| `Данные события`  | объект содержащий: *событие*, *уведомление*, *контекст*,
|                   | *отправитель события*, *флаг обработки события*,
|                   | *ссылка на результат* |

Требования
----------

- поддержка `std` и `no_std` для `Rust`
- можно использовать для проектов на языке `C`

Примеры
-------

Rust:
```rust
fn main() -> Result<(), Box<dyn Error>> {
    let app = App::new(&[MOD_MAIN])?;

    let result = app.run(Some(&"Context data"))?;

    Ok(())
}
```

Rust (no_std):
```rust
#[unsafe(no_mangle)]
fn main(argc: c_int, argv: *const *const c_char) -> c_int {
    let Ok(app) = App::new(&[MOD_MAIN], argc, argv) else {
        return libc::EXIT_FAILURE;
    };

    if let Err(_) = app.run(Some(&"Context data")) {
        return libc::EXIT_FAILURE;
    }

    libc::EXIT_SUCCESS
}
```

C:
```c
#include "libapp_kernel.h"

int main(int argc, const char *argv[]) {
    const AppModule modules[] = {MOD_MAIN};

    const App *app = app_new(modules, 1, argc, argv);
    app_run(app, &"Context data");
    app_free(app);

    return 0;
}
```
